### 3Way Handshake**(연결지향)**

**연결지향**을 보장하기 위해서 상대pc 접속을 요청하고 수락한 다음 데이터를 전송하고 전달 받았다는 확인을 받는다.3-1. TCP

1. **내 pc가 상대 pc에게 접속 요청을 함(SYN)**
    
    [내pc]——-(데이터 보내도 돼?)———>[상대pc]
    
2. **상대pc가 내pc에게 요청을 수락함(SYN+ACK)**
    
    [내pc]<——(응 보내도 돼)———[상대pc]
    
3. **그 다음 내pc가 요청을 응답하면서 데이터를 보낸다.(ACK)**
    
    [내pc]——(알았어 보낼게+[데이터])——>[상대pc]
    
4. **내pc가 데이터를 확인을 하면 상대 pc가 대답해줌(SYN+ACK)**
    
    [내pc]——(데이터 잘 갔어?)—————>[상대pc]
    
    [내pc]←——(응 잘 왔어)——————-[상대pc]
    
</aside>

<aside>
🤝

### 시퀀스 번호

패킷에 번호를 붙여서 받는 쪽에서 1)원래대로 순서를 조합하고  2)빠진 번호가 있으면 요청해서 패킷을 순서와 손상을 방지한다.

ex 1)

[내pc](패킷1,2,3)———(전송)———>[상대pc]

[내pc]———————(받음)——-—>[상대pc](패킷1,3,2)

패킷 순서대로 다시 조립  [상대pc](패킷1,2,3)

ex 2)

[내pc](패킷1,2,3)———(전송)———>[상대pc]

[내pc]———————(받음)——-—>[상대pc](패킷1,3)

[내pc]<——————(패킷2요청)——-—[상대pc](패킷1,3)

[내pc]——————(패킷2전달)——-—>[상대pc](패킷1,2,3)

# 5. DNS

<aside>
📌

## DNS필요 이유

우리가 웹사이트를 접속할때 해당 IP주소를 외워서 접속 할 수 없고 접속할려는 사이트의 IP주소가 변경 되었을때 또한 문제가 생긴다. 이를 방지 하기위해 DNS가 필요하다.

</aside>

<aside>
📌

## DNS란

도메인 네임 시스템(Domain Name System)으로 도메인 명을 IP주소로 변환하는 역할을 한다.

EX) 

“200.200.200.2”를 써야 google에 접속할 수 있지만 
도메인”google.com”을 통해서도 접속할 수 있게 만들어준다.

</aside>
